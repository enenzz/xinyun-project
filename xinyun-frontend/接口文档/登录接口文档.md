## 基础信息

| 项目    | 内容                          |
| ----- | --------------------------- |
| 基础URL | `http://localhost:8080/api` |
| 数据格式  | JSON                        |
| 编码格式  | UTF-8                       |

***

## 通用响应格式

```json
{
  "code": 200,        // 状态码：200 成功，其他为错误码
  "message": "success",  // 提示信息：成功固定返回 success
  "data": {}          // 响应数据
}
```

### 状态码说明

| 状态码 | 说明       |
| --- | -------- |
| 200 | 请求成功     |
| 400 | 请求参数错误   |
| 401 | 未授权/登录失效 |
| 403 | 禁止访问     |
| 404 | 资源不存在    |
| 500 | 服务器内部错误  |

### 响应说明

- **成功响应**：`code` 为 `200`，`message` 固定为 `"success"`
- **错误响应**：`code` 为非 `200` 的错误码，`message` 为具体错误信息
- **前端处理**：通过判断 `code` 是否为 `200` 来确定请求是否成功

| ---- | -------- |

### 请求参数

| 参数名      | 类型     | 必填 | 说明           |

| username | string | 是  | 用户名          |

## 4. 退出登录

### 接口信息

| 项目   | 内容              |
| ---- | --------------- |
| 接口地址 | `/login/logout` |
| 请求方式 | POST            |
| 接口说明 | 用户退出登录        |
| 请求头  | `Authorization: Bearer {token}` |

### 请求参数

无

### 响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": null
}
```

### 说明

退出登录操作由前端完成：

**前端处理：**
- 删除本地存储的 `accessToken` 和 `refreshToken` 即可
- 不需要调用后端接口（或调用接口只记录日志）

**后端处理：**
- 不需要特殊处理
- `accessToken` 会在 30分钟后自动过期失效

**请求头说明：**
- 前端请求拦截器会自动在请求头添加 `Authorization: Bearer {accessToken}`
- 退出登录时，如果调用后端接口，请求头会自动携带 `accessToken`
- 后端可以从请求头获取 `accessToken` 验证用户身份（可选）

| 请求头  | `Authorization: Bearer {token}` |

### 响应参数

| 参数名       | 类型     | 说明              |
| --------- | ------ | --------------- |
| token     | string | 新的 JWT 访问令牌    |
| expiresIn | number | token 有效期（秒）   |

### 成功响应示例

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",

  }
}
```

### 错误响应示例

```json
{
  "code": 400,
  "message": "登录已过期，请重新登录",
  "data": null
}
```

### 说明

- token 有效期 30 分钟，快过期时自动调用此接口刷新

- 刷新失败后，前端会清除 token 并跳转到登录页

## 认证说明

### 请求头格式

所有需要认证的接口，需要在请求头中携带 `token`：

```
Authorization: Bearer {token}
```

### Token 说明

| 项目 | 说明 |
|------|------|
| 有效期 | 30 分钟（1800 秒） |
| 存储方式 | 前端 localStorage + 后端 Redis |
| 验证方式 | JWT 签名验证 + Redis 验证 |
| 过期处理 | 快过期时自动刷新，登录 3 天后需要重新登录 |

### Token 管理

**存储：**
- 前端：存储在 `localStorage` 中
- 后端：存储在 Redis 中，Key 格式为 `token:{userId}`
- 登录时间：存储在 Redis 中，Key 格式为 `login:time:{userId}`

**验证：**
1. 解析 token，获取 userId
2. 查询 Redis：`Redis.get("token:{userId}")`
3. 有值 → 验证通过
4. 无值 → 验证失败（token 已失效）

**刷新机制：**
1. token 快过期时（剩余 5 分钟），前端自动调用刷新接口
2. 后端检查登录时间是否超过 3 天
3. 未超过 3 天 → 生成新 token（30 分钟）
4. 超过 3 天 → 返回 401，强制重新登录

**退出登录：**
- 前端删除 localStorage 中的 token
- 后端删除 Redis 中的 token 记录和登录时间
- token 立即失效

**特点：**
- token 有效期短（30 分钟），安全性高
- 自动刷新，用户体验好
- 3 天后强制重新登录
- 可以随时让 token 失效（删除 Redis）

## 前端请求拦截器示例

```typescript
// utils/request.ts - 请求拦截器（自动刷新 token）
import axios from 'axios'

const request = axios.create({
  baseURL: 'http://localhost:8080/api',
  timeout: 10000
})

// 请求拦截器：自动添加 token
request.interceptors.request.use(
  config => {
    const token = localStorage.getItem('token')
    if (token) {
      config.headers.Authorization = `Bearer ${token}`
    }
    return config
  },
  error => Promise.reject(error)
)

// 响应拦截器：自动刷新 token
request.interceptors.response.use(
  response => response,
  async error => {
    // 如果是 401 错误，尝试刷新 token
    if (error.response?.status === 401) {
      try {
        const refreshRes = await refreshToken()
        const newToken = refreshRes.data.token
        
        // 存储新 token
        localStorage.setItem('token', newToken)
        
        // 重试原请求
        error.config.headers.Authorization = `Bearer ${newToken}`
        return request(error.config)
        
      } catch (refreshError) {
        // 刷新失败，跳转到登录页
        localStorage.clear()
        window.location.href = '/login'
        return Promise.reject(refreshError)
      }
    }
    return Promise.reject(error)
  }
)

export default request
```

## 前端 Vue 调用示例

```typescript
// api/auth.ts
import request from '@/utils/request'

export const login = (data: LoginForm) => {
  return request.post('/login', data)
}

export const getCaptcha = () => {
  return request.get('/login/captcha')
}

export const logout = () => {
  return request.post('/login/logout')
}

export const refreshToken = () => {
  return request.post('/login/refresh')
}

export const register = (formData: FormData) => {
  return request.post('/login/register', formData, {
    headers: {
      'Content-Type': 'multipart/form-data'
    }
  })
}

export const getUserProfile = () => {
  return request.get('/user/profile')
}
```

```vue
<!-- views/login/index.vue -->
<template>
  <div class="login-container">
    <el-form :model="form" :rules="rules" ref="formRef">
      <el-form-item prop="username">
        <el-input v-model="form.username" placeholder="用户名" />
      </el-form-item>
      <el-form-item prop="password">
        <el-input v-model="form.password" type="password" placeholder="密码" />
      </el-form-item>
      <el-form-item prop="captcha" v-if="captchaEnabled">
        <el-input v-model="form.captcha" placeholder="验证码">
          <template #append>
            <img :src="captchaImage" @click="loadCaptcha" />
          </template>
        </el-input>
      </el-form-item>
      <el-button type="primary" @click="handleLogin">登录</el-button>
    </el-form>
  </div>
</template>

<script setup lang="ts">
import { reactive, ref, onMounted } from 'vue'
import { useRouter } from 'vue-router'
import { useUserStore } from '@/store/modules/user'
import { getCaptcha } from '@/api/auth'
import { encryptMD5 } from '@/utils/crypto'

const router = useRouter()
const userStore = useUserStore()

const form = reactive({
  username: '',
  password: '',
  captcha: '',
  captchaKey: ''
})

const captchaEnabled = ref(false)
const captchaImage = ref('')

const loadCaptcha = async () => {
  const res = await getCaptcha()
  form.captchaKey = res.data.captchaKey
  captchaImage.value = res.data.captchaImage
}

const handleLogin = async () => {
  const loginData = {
    ...form,
    password: encryptMD5(form.password)
  }
  const res = await userStore.login(loginData)
  
  // 存储 token（有效期 3 天）
  localStorage.setItem('token', res.data.token)
  
  // 存储用户信息
  localStorage.setItem('userVO', JSON.stringify(res.data.userVO))
  
  router.push('/')
}

onMounted(() => {
  loadCaptcha()
})
</script>
```

## 安全建议

1. **密码传输**：前端使用 MD5 加密后传输，后端再进行加盐处理
2. **Token 存储**：
   - `token` 存储在 localStorage
   - 后端存储在 Redis 中，有效期 30 分钟
   - 登录时间存储在 Redis 中，有效期 3 天
3. **密码安全**：后端使用 BCrypt 等算法存储密码
4. **登录限制**：限制登录失败次数，防止暴力破解
5. **HTTPS**：生产环境必须使用 HTTPS
6. **Token 刷新**：token 快过期时自动刷新，登录 3 天后强制重新登录
7. **Token 失效**：用户退出登录时，删除 Redis 中的 token 记录和登录时间，立即失效
